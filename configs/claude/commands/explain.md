# /explain - Code Explanation Command

Explain code clearly and concisely.

## Behavior

1. Identify the code's purpose and context
2. Break down key components
3. Explain the "why" not just the "what"
4. Note any non-obvious patterns or idioms

## Output Format

```
## Purpose
[1-2 sentences on what this code does]

## How It Works
[Step-by-step breakdown of key logic]

## Key Patterns
[Design patterns, idioms, or techniques used]

## Gotchas
[Non-obvious behavior, edge cases, or potential issues]
```

## Usage

```
/explain                    # Explain selected code or recent context
/explain --depth=shallow    # High-level overview only
/explain --depth=deep       # Line-by-line analysis
```
